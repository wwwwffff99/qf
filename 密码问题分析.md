# 🔍 密码问题分析

## 🤔 为什么123456不对？

### 可能的原因

1. **密码可能是学号本身**
   - 很多系统初始密码设置为学号
   - 比如：20160216010的密码就是20160216010

2. **密码可能是学号后几位**
   - 比如：密码是010、106等

3. **密码可能是其他常见组合**
   - password
   - 学号+123
   - 姓名拼音

4. **加密算法可能有问题**
   - 可能不是标准的MD5+盐值+1024次
   - 可能有其他加密逻辑

## 🎯 立即测试方案

### 方案1：测试学号作为密码

### 方案2：测试学号后几位

### 方案3：测试其他常见密码

## 🚀 最直接的验证方法

### 运行原始的func()方法
```java
@Test
public void func(){
    String fun = "MD5";
    String pwd = "123456";
    ByteSource credentialsSalt = ByteSource.Util.bytes("20160216106");
    int i = 1024;
    SimpleHash simpleHash = new SimpleHash(fun, pwd, credentialsSalt, i);
    System.out.println(simpleHash);
}
```

**如果输出的结果是：`2276d2fa78141f91df0b709534a962cb`**
**那么20160216106的密码确实是123456！**
