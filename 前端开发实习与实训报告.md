## 2 项目题目简介

### 2.1 项目背景

随着高校各类学科竞赛活动的日益增多，传统的线下报名方式已经难以满足高效管理的需求。学生需要通过纸质表格或邮件方式报名，管理人员需要手动整理报名信息，这种方式不仅效率低下，而且容易出错。因此，开发一套功能完善的竞赛报名管理系统具有重要的实际意义。

### 2.2 项目概述

竞赛报名管理系统是一个基于B/S架构的Web应用系统，采用前后端分离的开发模式。系统主要面向高校师生，提供竞赛信息发布、在线报名、队伍组建、报名管理等功能。

**前端部分**采用Vue.js框架开发，使用Element UI作为UI组件库，通过Axios与后端API进行数据交互。系统支持学生和管理员两种角色，不同角色拥有不同的功能权限。

## 3 项目需求分析与设计

### 3.1 市场调研

在开发本系统之前，我对市场上类似的竞赛报名系统进行了调研分析：

1. **高校教务系统**：大多数高校教务系统包含竞赛报名功能，但通常功能较为简单，仅支持个人报名，不支持组队功能。

2. **第三方竞赛平台**：如"学科竞赛网"等平台，功能较为完善，但主要面向全国性竞赛，对于校内竞赛的支持不够灵活。

3. **自建报名系统**：部分高校自建的报名系统，功能参差不齐，用户体验有待提升。

### 3.2 需求分析

通过调研分析，总结出本系统需要满足的核心需求：

#### 3.2.1 功能需求

1. **用户管理需求**
   - 支持学生和管理员两种角色
   - 实现安全的登录认证机制
   - 支持验证码校验防止恶意登录

2. **竞赛管理需求**
   - 支持发布不同级别的竞赛（院级、校级、省级、国家级）
   - 支持个人赛和组队赛两种类型
   - 支持设置报名截止时间
   - 支持上传竞赛相关附件

3. **报名管理需求**
   - 支持在线报名和取消报名
   - 支持组队报名，包括创建队伍、招募队员、申请加入
   - 支持导出报名名单为Excel文件

4. **信息发布需求**
   - 支持发布系统公告
   - 支持发布获奖通知
   - 支持文件附件上传下载

5. **数据统计需求**
   - 支持报名数据的统计分析
   - 支持数据可视化展示

#### 3.2.2 非功能需求

1. **易用性**：界面简洁美观，操作流程清晰
2. **响应性**：页面加载速度快，交互响应及时
3. **兼容性**：支持主流浏览器（Chrome、Firefox、Edge等）
4. **安全性**：防止XSS攻击、CSRF攻击等安全威胁

router.beforeEach((to, from, next) => {
  const type = to.meta.type
  if (type === 'login') {
    axios.get('/user/isLogin')
    .then((res) => {
      if (res.data.status == 1001) {
        next('/login')
      } else {
        next()
      }
    })
  } else {
    next()
  }
})

#### 4.2.2 登录功能实现

登录页面采用Element UI的表单组件，实现了以下功能：

1. **表单验证**：使用Element UI的表单验证规则
2. **验证码校验**：后端生成验证码图片，前端展示并校验
3. **角色选择**：支持学生和管理员两种角色登录
4. **记住登录状态**：通过Cookie保持登录状态

```vue
<template>
  <div class="login-container">
    <el-form :model="loginForm" :rules="rules" ref="loginForm">
      <el-form-item prop="username">
        <el-input v-model="loginForm.username" placeholder="请输入用户名">
          <i slot="prefix" class="el-icon-user"></i>
        </el-input>
      </el-form-item>
      <el-form-item prop="password">
        <el-input type="password" v-model="loginForm.password" placeholder="请输入密码">
          <i slot="prefix" class="el-icon-lock"></i>
        </el-input>
      </el-form-item>
      <el-form-item prop="code">
        <el-input v-model="loginForm.code" placeholder="验证码">
          <img slot="suffix" :src="codeUrl" @click="refreshCode" />
        </el-input>
      </el-form-item>
      <el-form-item>
        <el-radio-group v-model="loginForm.role">
          <el-radio label="2">学生</el-radio>
          <el-radio label="1">管理员</el-radio>
        </el-radio-group>
      </el-form-item>
      <el-button type="primary" @click="handleLogin">登录</el-button>
    </el-form>
  </div>
</template>

<script>
export default {
  data() {
    return {
      loginForm: {
        username: '',
        password: '',
        code: '',
        role: '2'
      },
      rules: {
        username: [{ required: true, message: '请输入用户名', trigger: 'blur' }],
        password: [{ required: true, message: '请输入密码', trigger: 'blur' }],
        code: [{ required: true, message: '请输入验证码', trigger: 'blur' }]
      },
      codeUrl: ''
    }
  },
  methods: {
    handleLogin() {
      this.$refs.loginForm.validate((valid) => {
        if (valid) {
          this.$axios.post('/user/login', this.loginForm)
            .then(res => {
              if (res.data.status === 200) {
                // 根据角色跳转不同首页
                if (this.loginForm.role === '1') {
                  this.$router.push('/adminHome')
                } else {
                  this.$router.push('/home')
                }
              }
            })
        }
      })
    },
    refreshCode() {
      this.codeUrl = '/user/getCode?' + new Date().getTime()
    }
  }
}
</script>
```

## 6 实训总结

### 6.1 技术收获

通过本次实训，我在以下几个方面获得了显著的技术提升：

1. **Vue.js框架掌握**：深入理解了Vue.js的核心概念，包括组件化开发、响应式数据绑定、生命周期钩子、计算属性、侦听器等。能够熟练使用Vue.js开发复杂的单页面应用。

2. **Element UI组件库使用**：熟练掌握了Element UI组件库的使用方法，包括表单组件、表格组件、对话框、消息提示等，能够快速构建美观规范的用户界面。

3. **前后端分离开发模式**：深入理解了前后端分离的开发模式，掌握了通过RESTful API与后端进行数据交互的方法，了解了跨域问题的解决方案。

4. **路由管理与权限控制**：学会了使用Vue Router进行路由管理，实现了基于路由守卫的登录权限控制。

5. **数据可视化技术**：学习了ECharts图表库的使用，能够实现常见的数据可视化需求。

### 6.2 项目经验

1. **需求分析能力**：通过参与项目需求分析，学会了如何从用户角度思考问题，如何将业务需求转化为技术实现。

2. **代码规范意识**：在开发过程中，注重代码的规范性和可维护性，养成了良好的编码习惯。

3. **问题解决能力**：在开发过程中遇到了各种技术问题，通过查阅文档、搜索资料、请教他人等方式，培养了独立解决问题的能力。

4. **团队协作经验**：在与后端开发人员的协作过程中，学会了如何进行有效的技术沟通，如何协调接口设计。
